<objective>
Add an admin authorization guard as the FIRST action in both `onboardClient` and `createClientLogin` Server Actions, before any service-role (`createAdminClient()`) client is constructed. Today these two actions call `createAdminClient()` (which bypasses RLS) with no caller check — any authenticated or anonymous visitor who can reach the Server Action can create client records, provision GHL sub-accounts, and mint Supabase Auth logins.

Purpose: Close SEC-01, a remotely exploitable privilege-escalation gap, so only authenticated admins can invoke admin-only write flows.
Output: Two Server Actions that reject non-admins before touching the service role.
</objective>
